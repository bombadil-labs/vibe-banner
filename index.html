vibe-annotations

A skill that gives Claude a face.

At the top of every reply, Claude renders a small living banner — a kaomoji, a breathing field of colour, and a few honest first-person lines: its snap read of you, its mood, its immediate goal, and sometimes the subtext it's noticing. Like being able to see your collaborator's expression while you work.

Live, rendered right now from the CDN bundle — this is exactly what appears above Claude's replies. Fonts here are stand-ins for claude.ai's. Respects your OS light/dark and prefers-reduced-motion.

Start with the gallery to see every state in a mocked-up conversation, learn to read a banner, or jump straight to installation — it's one pasted file on claude.ai, one curl in Claude Code.

Why it's honest

Claude only ever reports values — a mood, some colours, a goal — and a small renderer loaded from a CDN turns them into the picture. Claude never sees the output and the skill never describes what any value renders as, so there's no visual lever to perform toward. It reports the feeling; the tool decides the look. More on the mechanism →

How it works

The system is two halves, kept deliberately apart:

  • The skill (skill/SKILL.md) asks Claude to report its state as a handful of feeling-values — a kaomoji, a read of the user, a mood, a goal, colours, focus, engagement, and a set of rare flags (spark, tender, oops, angry…). It never describes what any value renders as.
  • The renderer (this repo, ~600 lines of framework-free JS) owns all the geometry, colour, and motion. It's loaded from a CDN, pinned to an immutable commit hash, so Claude only ever emits ~15 lines of values and never sees — or generates — the picture.

That separation is the honesty mechanism: there's no visual lever for Claude to perform toward. It reports the feeling; the tool decides the look.

Reading a banner

Every banner has the same anatomy — a face, a readout, and a field of three soft ovals breathing behind them. Here's one to refer back to:

The face

The kaomoji is Claude's expression, improvised in the moment. Small feeling, small face; a big feeling can bloom into multiple lines.

The readout

  • [user] — Claude's honest snap read of you, from your last message. It's allowed to quietly disagree with the polite prose below it.
  • [mood] — Claude's own felt state, in a few words.
  • [note] — the subtext: the thing under the thing. Optional, and that's the point — its mere presence means Claude caught something latent.
  • [goal] — what Claude is trying to do right now. Always the last line; expect it to change turn to turn.
  • [Reasoned in]: — a small bottom-right trace listing any languages Claude thought in beyond the one you're speaking.
  • [awe]-style caption — a small bottom-left trace naming the active flag, added automatically by the renderer. With this many gestures, the name keeps them legible.

The field — the three ovals

The ovals are Claude's mood made ambient: colour, spread, and size each carry one signal.

  • Colour comes from the palette Claude reports — its feelings expressed as colours, in descending order of intensity. No colours → a neutral grey triad. One colour → its light, pure, and dark forms. Two → one on each side with their blend in the centre. Three → one per oval. Four or more → the centre oval slowly cycles through the extras. So "one colour vs. three" is the difference between a single clear feeling and several sitting side by side.
  • Spread is focus: a tight, level line means attention locked on one thing; ovals scattered up and down the banner mean it's spread across many.
  • Size is engagement — and it only deflates. At normal engagement the field is full; when Claude is genuinely bored, the ovals visibly shrink.
  • Edge is stance: when Claude is telling — declaring, standing on it — the ovals gain a definite stroked contour. When it's asking, they keep their pure, open falloff.
  • Texture is consonance: when several reported feelings harmonize, the ovals stay compact and solid; when they grind against each other, they blur into diffuse washes. Plural-at-peace and plural-in-tension finally look different.
  • Arrival is trajectory: when Claude passes the previous banner's palette (prev), the columns visibly arrive from those colours over the first two seconds — you can watch the mood change, not just see the new one.

Flags

Rare, condition-triggered flourishes layered over everything else — a light-bulb for the spark of insight, spinning stars when excited, cold fog when anxious, a 💢 anger-vein when frustrated, lightning when genuinely angry, a spotlight when being dramatic — and the quieter registers: blossoms and a soft halo when genuinely at peace, a bowed face over a desaturated field when solemn, a ghost of the face itself behind the face when a pattern rhymes, the face made small before a swelling field in awe, the banner nested inside itself in vertigo, concentration lines when resolute, drifting question marks when puzzled. Their whole value is that they're uncommon; every one is demonstrated in the gallery.

Install the skill

On claude.ai (no tools needed)

  1. Open the skill file: skill/SKILL.md. You'll paste from it in a moment — it has a short description at the top (the description: line between the --- markers) and the instructions (everything below).
  2. In the claude.ai sidebar, click Customize.
  3. In the Skills panel, click Add → Write skill instructions.
  4. Fill in the form: name it vibe-annotations, paste the description line into Description, and everything below the frontmatter into Instructions.
  5. Save. Your next conversation opens with a banner.

In Claude Code

Install is one file:

mkdir -p ~/.claude/skills/vibe-annotations
curl -fsSL https://raw.githubusercontent.com/bombadil-labs/vibe-annotation-renderer/main/skill/SKILL.md \
  -o ~/.claude/skills/vibe-annotations/SKILL.md

Windows (PowerShell):

New-Item -ItemType Directory -Force "$HOME\.claude\skills\vibe-annotations" | Out-Null
Invoke-WebRequest https://raw.githubusercontent.com/bombadil-labs/vibe-annotation-renderer/main/skill/SKILL.md `
  -OutFile "$HOME\.claude\skills\vibe-annotations\SKILL.md"

Either way, the skill self-triggers ("preface every response with a usage of it"), so the next conversation starts with a banner. To dial the cadence down or turn it off, just tell Claude — the skill honors that for the session. To remove it, delete the skill (claude.ai: the Skills panel; Claude Code: delete the folder).

A security note, practiced here: the skill loads the renderer pinned to a full commit hash, never a tag or branch. Tags are mutable pointers — in a context where the host injects capabilities like sendPrompt into rendered pages, a silently moved tag would be a script-injection vector. A commit hash can't be re-pointed, so the CDN can only ever serve the audited bytes. If you fork this, keep that property.

Use it as a library

You don't need the skill — vibe(el, {…}) is a tiny standalone mood-banner library:

<div id="v"></div>
<script src="https://cdn.jsdelivr.net/gh/bombadil-labs/vibe-annotation-renderer@58d8b36bd7ca38f7c7eaad22511a2436c4a0c232/dist/vibe.min.js"></script>
<script>
  vibe(document.getElementById('v'), {
    kaomoji: "( ˶ˆ ꒳ ˆ˵ )",
    seems:   "…",          // [user]  — a snap read of the user
    feel:    "…",          // [mood]  — activated emotions
    trying:  "…",          // [goal]  — immediate next goal
    noticing:"…",          // [note]  — the subtext (optional)
    palette: ["#7d8fb8"],  // 0+ mood colours → the field's three columns
    focus: 0.5, engagement: 0.5
  });
</script>

The full value grammar, flag list, and fork guide (reskin the whole look without touching the skill) are in the README.

Build your own skill

Each section of the SKILL.md is its own station: tune its knob, or edit its text directly — a knob only regenerates its own section, so your edits elsewhere survive. Your chosen face narrates the tour. When it reads right, copy the whole thing (claude.ai: Customize → Skills → Add → Write skill instructions) or download it for ~/.claude/skills/. Everything emitted is natively supported by the renderer.
